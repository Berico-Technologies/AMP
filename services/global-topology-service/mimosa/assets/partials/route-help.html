Terminology

Routes are a construct of AMP. Routes represent the configuration necessary to produce or consumer messages (events) for a particular event type in a specific context. An event may actually have multiple routes, typically when events need to be forwarded to brokers in different clusters, or when you are using a more complex topology with virtual hosts. In general, however, it's wise to minimize the number of routes a client needs to connect.

Topics

Topics are aliases to routing keys used by exchanges. Topics are used to map the events that may have the same schema, but exist in different namespaces (or with languages with different namespace conventions than Java and C#).

Clients

The name or CN/DN of clients allowed to publish and consume on this route.

Producer Exchange

The exchange in which messages/events should be published to. It's ok if this is the same as the consumer exchange.

Consumer Exchange

The exchange in which messages/events should be consumed from. It's ok if this is the same as the producer exchange.
